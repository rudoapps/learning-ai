🤖 Agent vs 🎯 Skill vs 🖐️ Tool

Ladrillos, recetas y cocineros. La analogía perfecta para entender los tres niveles.

🧱 La analogía de la cocina

Tool = un utensilio. Un cuchillo, un horno, una sartén. Hace UNA cosa.

Skill = una receta. Combina varios utensilios en un orden concreto para lograr un plato. Es reutilizable.

Agent = un cocinero. Conoce recetas, tiene utensilios, pero además improvisa: si falta un ingrediente, busca un sustituto. Si algo se quema, ajusta. Decide por sí mismo qué hacer.

🖐️ Tool 🎯 Skill 🤖 Agent
Qué es Una función atómica Prompt + cadena de tools LLM + tools + bucle de decisión
Ejemplo send_email() "Analizar factura" (OCR → extraer → clasificar) "Reserva cena italiana" (busca, comprueba, se adapta, reserva)
Decide qué hacer No — le dicen qué ejecutar No — sigue un guion fijo Sí — decide en cada paso
Se adapta a fallos No No (o con lógica hardcoded) Sí — reintenta, cambia de estrategia
Coste 1 llamada al LLM N llamadas (una por paso) Variable (depende de iteraciones)
Predecibilidad Alta Alta Baja — puede sorprender
Complejidad Mínima Media Alta — depuración difícil

🌳 Árbol de decisión

❓ ¿La tarea requiere un solo paso o varios?
Un solo paso → Tool (get_weather, send_email, query_db)
❓ ¿Los pasos son siempre los mismos?
Sí, siempre → Skill (cadena fija, predecible, más barata)
❓ ¿Pueden surgir imprevistos o ramificaciones?
Sí → Agent (se adapta, pero pon guardrails)
No estoy seguro → empieza con Skill y escala a Agent si se queda corto

🏢 Ejemplo: gestión de tickets de soporte

Con Tool

classify_ticket(text) → devuelve categoría y urgencia. Una sola llamada. Para catalogar.

Con Skill

"Atender ticket": read_ticket → classify → draft_reply. Siempre los mismos 3 pasos. Para 80% de tickets estándar.

Con Agent

"Resuelve este ticket": lee el ticket → busca en la FAQ → si no encuentra, busca en JIRA → si es crítico, escala a un humano → si tiene solución, la aplica. Para tickets complejos que no siguen un guion fijo.

Heurística: si puedes escribir un script determinista para la tarea, no necesitas un agente. Empieza siempre por la opción más simple que funcione.
